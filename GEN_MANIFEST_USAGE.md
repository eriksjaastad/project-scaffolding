# scaffold gen-manifest

Auto-generates project metadata (purpose, entry points, commands, dependencies) and writes it into the project's `README.md` between markers. Re-runnable ‚Äî custom README content outside the markers is never touched.

## Usage

```bash
# Single project
uv run python -m scaffold.cli gen-manifest project-tracker

# With LLM-generated purpose line (calls local Ollama)
uv run python -m scaffold.cli gen-manifest project-tracker --refresh

# Preview without writing
uv run python -m scaffold.cli gen-manifest project-tracker --dry-run

# All scaffolded projects (any dir with .scaffolding-version)
uv run python -m scaffold.cli gen-manifest --all
uv run python -m scaffold.cli gen-manifest --all --refresh
```

Run from anywhere ‚Äî the command resolves paths relative to `$PROJECTS_ROOT`.

## What it scans

| Source | What it extracts |
|--------|-----------------|
| `pyproject.toml` | Project name, description, scripts (run command), pytest config |
| `package.json` | Name, description, npm scripts |
| Root + `src/` dirs | Entry point files: `main.py`, `app.py`, `server.py`, `cli.py`, `__main__.py`, `manage.py` |
| `CLAUDE.md`, `README.md`, configs | References to sibling projects ‚Üí internal dependencies |
| Local Ollama (`--refresh`) | One-line purpose sentence from `coding:current` model |

## What it writes

A block inside `README.md` between these markers:

```
<!-- MANIFEST:AUTO:START ‚Äî Auto-generated by `scaffold gen-manifest`. Do not edit this section. -->
**Purpose:** ...
**Status:** Active
**Entry points:** ...
**Key commands:** ...
**Dependencies (internal):** ...
**Last generated:** 2026-02-18
<!-- MANIFEST:AUTO:END -->
```

- **First run on a README with no markers:** Injects the block after the first `# heading`
- **Re-runs:** Replaces the existing block in place
- **No README exists:** Creates one with heading + block
- Everything outside the markers is preserved

## What it does NOT do

- Does not touch `AGENTS.md`, `CLAUDE.md`, or any AgentSync file
- Does not create a separate file (no `AI_MANIFEST.md`)
- Does not require a running daemon or external API
- Does not fail silently ‚Äî shows TODO placeholders when it can't detect something

## Code location

All implementation is in `scaffold/cli.py`:
- `_scan_pyproject()` ‚Äî reads pyproject.toml
- `_scan_package_json()` ‚Äî reads package.json
- `_scan_entry_points()` ‚Äî finds entry point files
- `_scan_internal_deps()` ‚Äî finds references to sibling projects
- `_llm_purpose()` ‚Äî calls local Ollama for purpose line
- `_build_manifest()` ‚Äî assembles the auto-generated content
- `_gen_manifest_for()` ‚Äî handles a single project (read/write/merge)
- `gen_manifest()` ‚Äî CLI command entry point
